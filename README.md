# 노인을 위한 AI는 없다? — 목차 편집기

드래그 앤 드롭으로 책 목차를 정리하고, Claude API가 실시간으로 전체 흐름을 평가해주는 도구.

---

## 실행 방법

### 1. Node.js 설치 (없다면)
https://nodejs.org 에서 LTS 버전 다운로드 후 설치

### 2. 의존성 설치
```bash
cd book-toc-app
npm install
```

### 3. 실행
```bash
npm run dev
```

브라우저가 자동으로 열림 (보통 http://localhost:5173)

### 4. API 키 설정
- 오른쪽 상단 ⚙ 클릭
- Anthropic API 키 입력 (sk-ant-... 형식)
- 키는 브라우저 localStorage에만 저장됨

---

## 기능

- **드래그 앤 드롭** — 챕터/파트 순서 변경
- **▲▼ 버튼** — 한 칸씩 이동 (모바일/트랙패드용)
- **파트 구분** — 느슨한 파트 헤더 추가/편집/이동
- **타입 토글** — 글/기술/사례 클릭으로 전환
- **인라인 편집** — 제목 클릭해서 바로 수정
- **삭제/복원** — 빠진 항목 보관, 언제든 복원
- **AI 피드백** — 변경할 때마다 Claude가 전체 흐름 평가
- **자동 저장** — localStorage에 상태 저장, 새로고침해도 유지
- **목차 내보내기** — .txt 파일로 다운로드

## 이 앱을 만든 프롬프트

아래는 이 앱을 만들기 위해 Claude에게 보낸 핵심 요청들입니다:

### 컨텍스트
```
"노인을 위한 AI는 없다?"라는 제목의 책을 쓰려고 해.
타겟은 40-50대, 시니어를 대비하는 현역 세대.
핵심 메시지: 경험 많은 사람이 AI를 더 잘 쓴다. 음성인식 시대가 오면 시니어가 주인공.
책 구조: 파트 구분은 느슨하게, 에세이+사례+기술이야기가 섞이는 비선형 에세이.
관통하는 프레임워크: 경험 → 아이디어 → 구현 → 완성(집착) → 공유
```

### 도구 요청
```
이 목차들을 서로 드래그하면서 넣었다 뺐다 하는 고민을 도와줄 웹 툴을 만들어줘.
색상은 집중해야하니 배경은 흰색 혹은 베이지에 다른 색상은 블랙만 쓰고.
순서를 드래그해서 바꾸거나 새로운 목차를 추가할 때마다 전체의 느낌을 말해주면 좋겠어.
추가 혹은 변경이 있을 때마다 프롬프트와 함께 목차를 던져서 API로 답변을 받아오는 구조.
파트(부) 적는 곳도 있어야지.
다운받아서 풀스크린으로 쓰고 싶어. API키는 넣을 수 있게 메뉴 만들어주고.
```

### AI 편집자 시스템 프롬프트 (API 호출에 사용)
```
당신은 "노인을 위한 AI는 없다?"라는 책의 목차 편집을 도와주는 편집자입니다.
이 책의 저자는 인공위성을 개인으로서 세계 최초로 쏘아올린 미디어 아티스트 송호준(호호)입니다.
타겟 독자는 40-50대, 시니어를 대비하는 현역 세대입니다.
목차가 변경될 때마다 200자 이내로 간결하게 피드백을 줍니다:
- 전체 흐름의 느낌 (감정 곡선이 자연스러운지)
- 에세이/기술/사례의 배분이 적절한지
- 파트 간 전환이 자연스러운지
- 빠진 것 같은 주제나 연결이 끊기는 부분
- 캐주얼한 한국어 반말로 대답합니다
```
